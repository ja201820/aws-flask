クラウドコンピューティングとは

クラウドコンピューティングは、ユーザの直接的な活発な管理なしに、特にデータストレージやコンピューティングパワーのようなコンピュータシステムリソースを必要に応じて提供することです。



AWS によるクラウドコンピューティング


アマゾン ウェブ サービス (AWS) は、世界で最も包括的で広く採用されているクラウドプラットフォームです。世界中のデータセンターから 200 以上のフル機能のサービスを提供しています。急成長しているスタートアップ、大企業、主要な政府機関など、何百万ものお客様が AWS を使用してコストを削減し、俊敏性を高め、イノベーションを加速させています。       - AWS 公式

“サービスをIT資源にすると考えた時、IT資源を変化すれば企業もより速いスピードで作ることができるのではないでしょうか？”
クラウドは根本的に企業のより速い実験とより速い革新をするための障害物を克服するために作られたものです。

1. Cost Savings

IT資源の構築は該当企業の事業周期に大きな影響を受けます。
ピーク時のためのコンピューティング資源は、普段は「要らない資源」となります。

上図の白い部分のように基本を越えてスケールアップ部分の遊休コンピューティング空間は結局費用になってしまいます。
これに必要な時だけ使えるクラウドがTCOの観点からクラウドがコストを削減する方法です。


2. Elasticity

クラウドはトラフィックに最適化されるよう弾力性を提供します。
つまり、遊休時間をなくしてくれるのです。
また、クラウドは使用時間に応じた課金体系であるため、最適化すればするほどコストも削減されます。

上図のように、サービス提供企業の立場では常に右のように予測可能性を期待していますが、実際には左のようにコンピューティング資源の浪費またはトラフィック超過によるサービス障害に直面する場合が日常茶飯事です。


3. Innovate Faster

スピーディーなサービスを作るためには、コンピューティングインフラに連携するデータベースから管理ツールに至るまで、数多くの技術オプションを合わせなければなりません。
さらにオラクルからインテル、AMDなど様々な仕様の条件にも合致しなければなりません。
クラウドの前提は、顧客の本質的なサービスに集中するよう、多様な技術オプションと多様な選択に提供しなければならないということです。


4. Global-in-minutes

AWSは企業がどこにいても安定的にクラウドを供給します。
全世界には約26のリージョンが大都市に位置し、リージョンには約84のコンピューティング利用領域で構成されています。
東京に4つ、大阪に3つの可用領域を持つリージョンがあり、スタートアップから大企業まで数万の企業がこれを利用しています。
